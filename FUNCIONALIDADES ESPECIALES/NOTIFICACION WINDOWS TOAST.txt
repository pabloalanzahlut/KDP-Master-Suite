## ⚠️ LIMITACIONES ACTUALES (Por entorno)

### 1. **winotify requiere Windows**
**Problema**: La librería winotify solo funciona en Windows, no en macOS/Linux.

**Estado actual**:
- `_init_native()` retorna `None` si falla import
- Toast interno funciona en todas las plataformas

**Solución**: El fallback está implementado; en mac/Linux solo se muestran toasts internos.

### 2. **Action buttons en winotify pueden variar según Windows version**
**Problema**: Los botones de acción en notificaciones nativas dependen de la versión de Windows.

**Estado actual**:
- Código usa `winotify.Action("Abrir App", callback)`
- Puede no funcionar en Windows 10 Home edition

**Solución**: El callback es opcional; si falla, la notificación se muestra sin acción.

### 3. **Cooldown global vs por tipo**
**Problema**: El cooldown actual es por combinación type+title, no considera otros factores.

**Estado actual**:
- cooldown_key = f"{type}_{title.replace(' ', '_')}"
- Para 129 canales, puede bloquear notificaciones legítimas de diferentes canales

**Solución**: En futuras versiones, cooldown por categoría (video_new, error, success) sin título.

## 📁 NUEVOS MÉTODOS Y CAMPOS

### NotificationRouter (notification_router.py)

**Clase principal**:
```python
class NotificationRouter:
    def __init__(self, app_instance):
        self.app = app_instance
        self.settings = getattr(app_instance, 'settings', {}) or {}
        self.notif_settings = self.settings.get("notifications", {})
        self.cooldown_seconds = self.notif_settings.get("cooldown_minutes", 5) * 60
        self.cooldowns = {}
        self.native_notifier = self._init_native()   # winotify.Winotify
        self.internal_notifier = self._init_internal()  # ToastNotification
        self._root = getattr(app_instance, 'root', app_instance)

    def _init_native(self) -> Optional[winotify.Winotify]
    def _init_internal(self) -> Optional[ToastNotification]
    def _schedule_after(delay_ms, callback)
    def notify(title: str, message: str, type: str = "info", action: Callable = None)
    def _should_cooldown(title: str, type: str) -> bool
    def _update_cooldown(title: str, type: str)
    def _show_internal_notification(title, message, type, action)
    def _show_native_notification(title, message, type, action)
    def _handle_native_click(action: Callable)
    def _log(message: str, level: str)
```

**Configuración (settings.json)**:
```json
{
  "notifications": {
    "enable_native": false,
    "enable_internal": true,
    "cooldown_minutes": 5,
    "summary_mode": false
  }
}
```

**Flujo de notificación**:
```
1. Servicios/UI llaman: notifier.notify("Título", "Mensaje", "info", callback)
   ↓
2. ¿Cooldown activo para este type+title? → SÍ → Omitir notificación
   ↓ NO
3. ¿enable_internal=True? → SÍ → Mostrar toast Tkinter (thread-safe)
   ↓
4. ¿enable_native=True? → SÍ → Mostrar notificación Windows Action Center
   ↓
5. Click en notificación nativa → action callback ejecuta en hilo principal
```

### Integración en monitor_tab.py

```python
# Callbacks ahora usan notifier
def on_monitor_new_video(self, video):
    self.log_monitor(f"✨ Nuevo video: {video.get('title', 'Sin título')}")
    if hasattr(self, 'notifier') and self.notifier:
        self.notifier.notify(
            title="Nuevo Video Detectado",
            message=video.get('title', 'Sin título')[:60],
            type="info",
            action=lambda: self.root.after(0, lambda: self.notebook.select(self.tab_monitor))
        )

def on_monitor_processing_complete(self, video):
    if hasattr(self, 'notifier') and self.notifier:
        self.notifier.notify(
            title="Video Procesado",
            message=video.get('title', 'Sin título')[:60],
            type="success"
        )

def on_monitor_error(self, error):
    if hasattr(self, 'notifier') and self.notifier:
        self.notifier.notify(
            title="Error en Monitor",
            message=str(error)[:80],
            type="error"
        )
```
