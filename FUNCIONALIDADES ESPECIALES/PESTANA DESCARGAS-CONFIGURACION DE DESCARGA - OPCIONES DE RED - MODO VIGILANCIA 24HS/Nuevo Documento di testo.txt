✅ **¿TIENE SENTIDO ASÍ?** Sí. Separar la capa determinista (conexión, estabilidad, monitoreo) de la capa cognitiva (predicción, optimización, diagnóstico) garantiza que el Modo Vigilancia 24h opere de forma continua sin saturar recursos, adaptándose dinámicamente a la red y asegurando la ingesta de texto. La arquitectura cumple estándares de disponibilidad empresarial y soberanía de datos local.

📦 **LISTA 1: 60 MÓDULOS SIN IA (Infraestructura Determinista & Control de Red)**

1. **Validador de Latencia Continua**: Mide el tiempo de respuesta del servidor cada 60s para detectar degradación.
2. **Rotación de Proxy Automática**: Cambia de nodo proxy si la latencia supera un umbral configurable.
3. **Gestor de Reintentos con Backoff Exponencial**: Reintenta peticiones fallidas con intervalos crecientes para evitar baneos.
4. **Limitador de Ancho de Banda Dinámico**: Restringe el consumo de red en horas pico según política corporativa.
5. **Sincronizador de Hora (NTP)**: Asegura que los timestamps de ingesta coincidan con la fuente para evitar duplicados.
6. **Verificador de Disponibilidad de DNS**: Valida la resolución de dominios antes de iniciar ciclos de vigilancia.
7. **Buffer de Peticiones Asíncrono**: Cola peticiones de red para procesarlas sin bloquear el hilo principal.
8. **Monitor de Consumo de Recursos**: Alerta si el hilo de vigilancia supera el 5% de CPU o RAM.
9. **Gestión de Cookies/Sesión Persistente**: Mantiene sesiones activas para evitar re-logins en fuentes privadas.
10. **Detector de Caída de Red**: Pausa la vigilancia automáticamente si se pierde la conectividad y reanuda al recuperarla.
11. **Registro de Actividad de Red (Log)**: Log detallado de cada petición HTTP, respuesta y estado para auditoría.
12. **Cifrado de Datos en Tránsito (TLS 1.3)**: Fuerza conexión segura para proteger las transcripciones en tránsito.
13. **Validador de SSL**: Verifica certificados del servidor de origen antes de descargar datos.
14. **Throttling por Horario**: Ajusta la velocidad de descarga según franjas horarias definidas por el usuario.
15. **Recuperación de Estado de Red**: Restaura la configuración de red previa tras un fallo crítico.
16. **Priorización de Peticiones de Subtítulos**: Da prioridad a la descarga de `.vtt/.srt` sobre otros metadatos.
17. **Limpieza de Cache de DNS**: Limpia cache si se detectan errores de resolución persistentes.
18. **Validador de Respuesta HTTP**: Verifica códigos 200 OK antes de procesar la respuesta como válida.
19. **Gestor de Timeouts Configurable**: Permite definir tiempos máximos de espera por tipo de fuente.
20. **Balanceador de Carga de Descarga**: Distribuye las peticiones entre múltiples hilos de red si está habilitado.
21. **Aislamiento de Hilo de Vigilancia**: Ejecuta el monitor en un proceso separado para no afectar la UI.
22. **Protocolo de Desconexión Segura**: Cierra conexiones abiertas limpiamente al detener el servicio.
23. **Persistencia de Configuración de Red**: Guarda ajustes de proxy y límites en un archivo cifrado.
24. **Validador de Formato de Transcripción**: Verifica que el archivo descargado sea texto válido antes de guardarlo.
25. **Detección de Bloqueo de IP**: Identifica respuestas 403 y sugiere cambio de IP automáticamente.
26. **Gestor de User-Agent**: Rota identificadores de navegador para simular comportamiento humano.
27. **Validador de Tamaño de Archivo (Texto)**: Descarta archivos vacíos o excesivamente grandes que puedan ser basura.
28. **Monitor de Integridad de Datos Descargados**: Verifica que el archivo `.txt` tenga contenido legible.
29. **Sistema de Alarma de Fallo**: Notifica al usuario si la vigilancia falla más de 3 veces consecutivas.
30. **Gestor de Excepciones de Red**: Captura y maneja errores de socket sin cerrar la aplicación.
31. **Optimizador de Conexión TCP**: Ajusta parámetros de ventana de red para maximizar throughput.
32. **Verificador de Espacio en Disco Continuo**: Pausa la vigilancia si el disco de destino está lleno.
33. **Compresor de Logs de Vigilancia**: Comprime logs antiguos para ahorrar espacio de almacenamiento.
34. **Gestor de Prioridades de Cola**: Permite definir qué canales se vigilan primero en la cola.
35. **Validador de Certificados Root**: Verifica la cadena de confianza de los certificados recibidos.
36. **Sistema de Fallback de DNS**: Usa DNS alternativos si el primario falla.
37. **Monitor de Uso de Memoria**: Libera memoria caché de red si el consumo supera el límite.

40. **Sistema de Auditoría de Acceso**: Registra quién modificó la configuración de red.
41. **Gestor de Errores 429**: Pausa automáticamente al detectar límite de peticiones por segundo.
42. **Validador de Cabeceras HTTP**: Verifica que las respuestas incluyan headers requeridos.
43. **Monitor de Latencia de API**: Mide el tiempo de respuesta de las APIs de terceros.
44. **Gestor de Conexiones Simultáneas**: Limita el número de hilos activos de red para no saturar.
