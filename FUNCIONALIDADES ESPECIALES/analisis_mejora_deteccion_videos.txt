## ⚠️ LIMITACIONES ACTUALES (Por entorno)

### 1. **Filtrado por fecha puede perder videos históricos**
**Problema**: Si `max_age_days` es muy bajo, videos de hace más de 7 días nunca se detectan.

**Estado actual**:
- Por defecto: 7 días
- Configurable: Sí (método `set_detection_config()`)

**Solución**: El usuario puede ajustar `max_age_days` según necesidad

### 2. **Detección de reposts por título puede tener falsos positivos**
**Problema**: Títulos similares no siempre son reposts (pueden ser serieslegítimas).

**Estado actual**:
- Similitud threshold: 85%
- Método: Comparación de palabras (Jaccard)

**Solución**: Se marca como "posible repost" pero se procesa igual

## 📁 NUEVOS MÉTODOS Y CAMPOS

### Base de Datos (db_manager.py)

**Tabla channels** - Nuevos campos:
```sql
priority INTEGER DEFAULT 3  -- 1=baja, 5=alta prioridad
```

**Tabla videos** - Nuevos campos:
```sql
content_hash TEXT          -- MD5 del contenido limpio
is_repost BOOLEAN DEFAULT 0  -- Marcador de posible repost
```

**Nuevos métodos**:
- `update_channel(channel_id, priority=N)` - Actualizar prioridad
- `get_all_channels(active_only, priority_filter)` - Filtro por prioridad
- `get_priority_channels()` - Canales ordenados por prioridad
- `update_video_content_hash(video_id, content_hash)` - Guardar hash post-descarga
- `check_content_hash_exists(content_hash)` - Verificar duplicado
- `check_title_similarity(channel_id, title, threshold)` - Detectar reposts
- `mark_as_repost(video_id)` - Marcar video como repost

### Monitor Service (monitor_service.py)

**Nuevas propiedades**:
```python
self.max_results_per_check = 50   # Videos por canal
self.max_age_days = 7              # Ignorar mayores a 7 días
self.enable_video_cache = True    # Caché de video_ids
self.video_cache = {}             # {channel_id: set(video_ids)}
```

**Nuevos métodos**:
- `set_detection_config(max_results, max_age_days)` - Configuración dinámica
