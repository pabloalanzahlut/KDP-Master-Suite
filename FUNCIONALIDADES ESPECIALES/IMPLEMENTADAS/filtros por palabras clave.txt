## ⚠️ LIMITACIONES ACTUALES (Por entorno)

### 1) Formato de extracción de metadatos de YouTube
- Problema: Las palabras clave pueden depender de metadatos (p. ej., tags) que no siempre están disponibles en modo rápido (extract_flat=True).
- Estado: El filtrado opera sobre título y descripción; el extraído de tags/IA se usa solo cuando está disponible.
- Solución propuesta: Hacer llamadas adicionales solo si hay coincidencias parciales en duración/ventana de tiempo, para luego aplicar tags si están presentes.

### 2) Semántica avanzada opcional
- Problema: La similitud semántica (IA) requiere un servicio IA disponible.
- Estado: Capa IA disponible solo si el servicio está configurado; si no, se usa flujo sin IA.
- Solución propuesta: Mantener un fallback completamente local y permitir activar IA solo cuando esté disponible.

### 3) Diálogo de decisión y flujo de usuario
- Problema: El flujo de decisión para duplicados (esp. en UI) debe enlazarse con el diálogo modal.
- Estado: El diálogo está preparado, pero requiere conexión completa a UI para activar automáticamente.
- Solución propuesta: Enlazar DuplicateDecisionDialog con el flujo de monitor para permitir decisiones automáticas o manuales según configuración.

## 🎯 NUEVOS MÉTODOS Y CAMPOS

### KeywordFilter (app/core/keyword_filter.py)
- Métodos clave:
  - __init__(include_keywords=None, exclude_keywords=None, mode="OR") – inicializa filtros.
  - should_process(title, description="") -> (bool, str) – decide passage del filtro y devuelve razón.
  - to_dict() / from_dict(data) – serialización/deserialización.
  - to_json() / from_json(json_str) – serialización JSON.
  - test_filter(test_title, test_description="") – prueba rápida del filtro.
  - parse_keywords(keywords_str) – parsea cadenas separadas por coma.
- Campos clave:
  - include_keywords, exclude_keywords (listas en minúsculas)
  - mode ("OR" | "AND")
  - enabled (boolean)

### DB Manager (app/database/db_manager.py)
- Nuevos métodos:
  - save_keyword_filter(include_keywords, exclude_keywords, mode, enabled) -> bool
  - get_keyword_filter() -> Optional[Dict]
  - increment_ignored_count(count=1) -> bool
  - get_filter_statistics() -> Dict
- Nueva estructura de DB:
  - Tabla keyword_filters con campos include_keywords (JSON), exclude_keywords (JSON), mode, enabled, total_ignored, ignored_today, last_reset_date, etc.
- Migraciones:
  - Añadida tabla keyword_filters
  - Añadidas columnas en videos: ignored_by_filter, filter_reason (para rastreo de por qué fue ignorado)
  - Índices relevantes para rendimiento

### Monitor Service (app/services/monitor_service.py)
- Capacidades añadidas:
  - Instancia de KeywordFilter y carga desde BD al inicio.
  - Método set_keyword_filter(include_keywords, exclude_keywords, mode, enabled) para persistencia y actualización.
  - Método get_keyword_filter_config() para exponer la configuración actual.
  - Integración en _check_single_channel para aplicar should_process sobre cada video detectado.
  - Conteo de videos ignorados incrementando el contador de filtros en la BD.
  - Prueba de filtrado (test_keyword_filter) para validaciones en UI.

### UI (gui_app.py)
- Panel dedicado "Filtros por Palabras Clave" dentro del monitor, con:
  - Opción para habilitar/deshabilitar
  - Campos para incluir y excluir (listas separadas por coma)
  - Selección de modo OR/AND
  - Botones Guardar / Probar / Reset
  - Indicador de estadísticas del filtro
- Métodos en GUI para cargar/guardar configuración desde/para el monitor, validar entradas y mostrar resultados de pruebas.
