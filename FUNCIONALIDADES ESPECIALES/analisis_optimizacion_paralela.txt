## ⚠️ AJUSTES CRÍTICOS (Obligatorios)

### 1. **Deduplicación Thread-Safe**
**Problema**: El hash MD5 global se verifica en un diccionario compartido. En paralelo, dos hilos pueden verificar el mismo hash al mismo tiempo → **duplicados procesados**.

**Solución quirúrgica**:
```python
import threading

# En __init__ de ProcessingService:
self._hash_lock = threading.Lock()
self._processed_hashes = set()

# En process_single_file (dentro del lock):
with self._hash_lock:
    if file_hash in self._processed_hashes:
        return {'status': 'duplicate', 'hash': file_hash}
    self._processed_hashes.add(file_hash)
```

### 2. **Orden de Callbacks (Opcional pero recomendado)**
**Problema**: Los callbacks pueden llegar en orden aleatorio, rompiendo la progresión visual ("Procesando 3/100" antes que "2/100").

**Solución mínima**:
```python
# Usar queue para preservar orden de inicio (no de finalización)
from queue import Queue

# En lugar de llamar callback directamente:
self.callback_queue.put({'index': file_index, 'result': result})

# Hilo separado que consume la queue en orden:
def _callback_dispatcher(self):
    while True:
        item = self.callback_queue.get()
        if item is None: break  # Sentinel para terminar
        self._original_callback(item['index'], item['result'])
```

## 🎯 RECOMENDACIÓN FINAL

**Usa `ThreadPoolExecutor`** (no ProcessPoolExecutor) porque:
- Tu carga es **I/O-bound** (leer/escribir archivos), no CPU-heavy.
- `ThreadPool` comparte memoria → más fácil para el lock de deduplicación.
- Menor overhead que `ProcessPool` (sin pickle/unpickle de objetos).

**Número de workers**: `min(32, os.cpu_count() * 4)` (recomendación oficial de Python para I/O).
