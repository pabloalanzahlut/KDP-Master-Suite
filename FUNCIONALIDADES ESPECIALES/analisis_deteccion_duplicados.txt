## ✅ PUNTOS FUERTES (Mantenidos)

| Punto | Porqué es correcto |
|-------|---------------------|
| **Estrategia híbrida escalonada** | Hash → Duration → Title → Tags → IA (orden de menor a mayor coste) |
| **Optimización de rendimiento** | Ventana de 7 días limita comparación de 3000 a ~50 videos |
| **Tag Fingerprinting** | Comparar tags es O(1) vs O(n) de IA, alta precisión para mismo nicho |
| **Fallback graceful** | Si API IA no disponible, usa métodos locales sin romper |
| **Original vs Repost** | Lógica de negocio correcta: descartar nuevo, conservar antiguo |
| **UI de decisión interactiva** | Usuario decide acción (procesar ambos/omitir/agrupar/ignorar siempre) |
| **Tabla de relaciones** | video_relations permite trackear relaciones entre duplicados |
| **Migration DB retrocompatible** | ALTER TABLE con TRY/EXCEPT no rompe instalaciones existentes |
| **Índices optimizados** | Índices en duration_seconds, published_at aceleran queries |
| **Integración con Monitor** | Detector integrado en flujo de detección de nuevos videos |

## ⚠️ LIMITACIONES ACTUALES (Por entorno)

### 1. **Tags con extract_flat=True no disponibles**
**Problema**: Los tags de YouTube no están disponibles en modo rápido (extract_flat=True).

**Estado actual**:
- Monitor usa extract_flat=True para reducir cuota API
- Tags solo disponibles con llamada adicional no-flat

**Solución**: Hacer llamada adicional solo cuando hay match parcial de duración + ventana de tiempo

### 2. **Similitud semántica requiere API IA**
**Problema**: Para similitud semántica (capa 5) se necesita embedding vectorial.

**Estado actual**:
- Default: Solo capas 1-4 (hash, duración, título, tags)
- Capa 5 (IA): Solo si AI wrapper disponible y configurado

**Solución**: Fallback a métodos locales si API no disponible

### 3. **Diálogo de decisión no conecta aún con callback**
**Problema**: El diálogo está creado pero no se activa automáticamente en el flujo.

**Estado actual**:
- DuplicateDecisionDialog creado y funcional
- Callback on_duplicate_detected configurado en monitor
- UI muestra notificaciones toast pero no diálogo modal

**Solución**: Conectar callback en gui_app.py para mostrar diálogo cuando usuario configure preferencia

### DuplicateDetector (duplicate_detector.py)

**Nuevas clases principales**:
```python
class DuplicateType(Enum):
    EXACT_CONTENT = "exact_content"       # Mismo hash de contenido
    REPOST = "repost"                     # Título similar (repost)
    SIMILAR_DURATION = "similar_duration" # Duración similar + ventana de tiempo
    SIMILAR_TAGS = "similar_tags"         # Tags en común
    SEMANTIC_SIMILAR = "semantic_similar" # Similitud semántica por IA

class DuplicateLevel(Enum):
    HIGH = "high"       # >90% seguro
    MEDIUM = "medium"   # 60-90% seguro
    LOW = "low"        # <60% seguro

class DuplicateDetector:
    DEFAULT_CONFIG = {
        'duration_tolerance': 5,      # segundos
        'time_window_days': 7,         # días hacia atrás
        'title_threshold': 0.80,       # 80% similitud
        'min_common_tags': 3,          # mínimo tags en común
        'tags_window_days': 30,        # ventana para tags
        'ia_threshold': 0.75,          # umbral para usar IA
        'max_ia_comparisons': 5,        # máximo comparaciones IA por video
    }
    
    def check_duplicate(video_info: Dict) -> Tuple[bool, Optional[Dict]]
    def _check_exact_duplicate(content_hash: str) -> bool
    def _check_duration_window(duration: int) -> List[Dict]
    def _check_title_similarity(title: str, channel_id: int = None) -> List[Dict]
    def _check_tags_match(tags: List[str]) -> List[Dict]
    def _check_semantic_similarity(content_preview: str, max_results: int = 5) -> Optional[Dict]
    def _cosine_similarity(vec1: List[float], vec2: List[float]) -> float
    def get_detection_summary(video_info: Dict) -> Dict
    def clear_cache()
```

**Flujo de detección**:
```
1. Video detectado en canal
   ↓
2. ¿Hash contenido existe? → SÍ → [A] Duplicado exacto (HIGH)
   ↓ NO
3. Similitud título >80% → SÍ → [B] Repost (MEDIUM)
   ↓ NO
4. Duration similar (±5s) + ventana 7 días → SÍ → [C] Similar Duration (MEDIUM)
   ↓ NO
5. Tags en común ≥3 → SÍ → [D] Similar Tags (MEDIUM)
   ↓ NO
6. ¿API IA disponible? → SÍ → Embeddings → Similitud semántica → [E] Semantic (LOW)
   ↓ NO
7. Procesar normalmente
```
